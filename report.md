# Отчёт о реализации компилятора BestLang

## Архитектура

Компилятор реализован как линейный pipeline из независимых фаз:

```
source (.bl)
  │
  ▼ Lexer::tokenize()          → std::vector<Token>
  │
  ▼ Parser::parse()            → Program AST
  │
  ▼ Semantic::analyze()        → AnalysisResult (аннотированный AST)
  │
  ▼ IR::lower()                → IRProgram (трёхадресный код)
  │
  ▼ IR::optimize()             → оптимизированный IRProgram
  │
  ▼ Codegen::compile()         → std::vector<uint8_t>  (.blc байткод)
  │
  ▼ VM::run_bytecode()         → результат выполнения
```

Каждая фаза реализована в отдельном модуле со своим namespace.

## Фазы компиляции

### 1. Лексический анализатор (`src/lexer.hpp/cpp`)

- Ручной однопроходной лексер (без генераторов)
- Поддерживает: десятичные, шестнадцатеричные (`0x`), двоичные (`0b`) целые литералы с суффиксами типов
- Вещественные литералы с экспонентой и суффиксами `f32`/`f64`
- Строковые литералы с escape-последовательностями (`\n`, `\t`, `\r`, `\"`, `\\`, `\0`, `\xHH`)
- Разделитель разрядов `_` в числовых литералах
- Формат диагностики: `file:line:col: error: message`

### 2. Синтаксический анализатор (`src/parser.hpp/cpp`)

- Рекурсивный спуск (Recursive Descent)
- Вычисление приоритетов операторов через таблицу precedence climbing
- Строит AST с помощью иерархии классов и `std::unique_ptr`
- Нечепочные операторы сравнения: `a < b < c` — синтаксическая ошибка
- Поддерживаются: структурные литералы, литералы массивов, l-value произвольной глубины

### 3. Семантический анализатор (`src/semantic.hpp/cpp`)

Двухпроходной анализ:
- **Pass 1**: сбор всех top-level деклараций (функции, структуры, глобальные переменные, псевдонимы типов, пространства имён)
- **Pass 2**: проверка тел функций

Проверки:
- Разрешение имён (лексическая область видимости)
- Проверка типов (номинальная типизация, нет неявных приведений)
- Неизменяемость `val`-переменных
- Достижимость всех путей возврата для non-void функций
- `break`/`continue` только внутри циклов
- Обязательная инициализация переменных при объявлении

### 4. Промежуточное представление (IR) (`src/ir.hpp`, `src/lowering.cpp`)

Трёхадресный код:
- Временные переменные (`t0`, `t1`, ...)
- Разделение на целочисленные, вещественные и логические операции
- Явные инструкции для массивов, структур, строк
- Jump с метками + backpatching для short-circuit вычислений

### 5. Оптимизатор (`src/optimizer.cpp`)

Constant folding:
- Сворачивание выражений с двумя константными операндами на этапе компиляции
- `2 + 3` → `5`, `true and false` → `false`

### 6. Генератор кода (`src/codegen.hpp/cpp`)

Целевая платформа: **BL-VM** (байткод формата `.blc`).

Структура файла:
- Заголовок 24 байта: магическое число `0x424C`, версия `0x01`, смещения секций
- Пул констант с дедупликацией (int64, uint64, float64, bool, string, имена функций)
- Секция функций (имя, число параметров, число слотов, код)
- Секция глобальных переменных (индексы инициализаторов в пуле констант)
- Таблица отладочных строк

Трансляция IR → байткод:
- Каждое IR-значение материализуется через LOAD/STORE в слоты вызова
- Jump backpatching для разрешения меток
- Отдельные опкоды для каждой операции по типу

### 7. Виртуальная машина (`src/vm.hpp/cpp`)

Стековая архитектура BL-VM:
- Стек значений с тегами типов (`Value::Kind`)
- Таблица вызовов с фреймами: слоты локальных переменных
- Соглашение о вызовах: аргументы слева направо перед `CALL`, результат на стеке вызывающего
- Реализованы все опкоды: арифметика, сравнения, логика, массивы, структуры, строки, приведения типов
- Встроенные функции: `print` (любой тип), `input`, `exit`, `panic`
- Runtime-ошибки: деление на ноль, выход за границы массива, неверное приведение NaN/Inf к целому

## Обоснование решений

| Решение | Обоснование |
|---------|-------------|
| Рекурсивный спуск | Прост в реализации, легко сопоставляется с грамматикой |
| `std::unique_ptr` для AST | Чёткое владение, нет утечек памяти |
| `std::variant` для IR | Безопасное сопоставление с образцом без RTTI |
| `std::expected`-style через `std::variant` | Явная обработка ошибок без исключений |
| Отдельный модуль для VM | Позволяет использовать VM независимо от компилятора |
| Таблица приоритетов в парсере | Легко расширяется, без левой рекурсии |

---

## Ограничения

- Таблица отладочных строк не заполняется (нет line numbers в runtime)
- Глобальные переменные поддерживают только константные инициализаторы
- Нет многофайловой компиляции (один файл = одна программа)
- Float-операции не проверяют NaN при арифметике (только при `as int`)

## Сборка и запуск

```bash
make build          # компилятор myc + VM blvm
./myc prog.bl --run # скомпилировать и выполнить
./myc prog.bl -o prog.blc && ./blvm prog.blc  # раздельно
./myc prog.bl --dump-tokens   # токены
./myc prog.bl --dump-ast      # AST
./myc prog.bl --dump-ir       # трёхадресный IR
```

## Дополнительные возможности

- `--dump-ir`: вывод трёхадресного промежуточного представления
- `--run`: немедленный запуск скомпилированного байткода через встроенную VM
- `blvm <file.blc>`: автономный исполнитель байткода
- Constant folding оптимизация в IR-фазе

# BestLang — компилятор языка программирования

Учебный проект: собственный язык программирования и компилятор к нему, написанный на C++23.

## Язык BestLang

BestLang — статически типизированный императивный язык с синтаксисом, вдохновлённым Python и Java. Компилируется в байт-код собственной стековой виртуальной машины BL-VM.

### Возможности языка

- **Типы:** `int8`..`int64`, `uint8`..`uint64`, `float32`, `float64`, `bool`, `string`, `void`
- **Составные типы:** массивы фиксированного размера `[T; N]`, динамические массивы `[T]`, структуры `struct`, кортежи `(T1, T2)`
- **Переменные:** `var` (мутабельные) и `val` (иммутабельные) с выводом типа
- **Управление потоком:** `if/else`, `while`, `for i in 0..n`, C-style `for`, `break`, `continue`, `return`
- **Функции:** параметры, возврат значений, рекурсия, перегрузка (overloading)
- **Структуры и методы:** `struct` + `impl` для добавления методов
- **Пространства имён:** `namespace` с доступом через `::`
- **Псевдонимы типов:** `type Meters = float64`
- **Nullable типы:** `int32?`, значение `null`
- **Неявное расширение:** `int8 + int32` → автоматически `int32`
- **Явное приведение:** `expr as T`
- **Строки:** UTF-8, конкатенация `+`, длина `.len()`, сравнение `==`/`!=`
- **Встроенные функции:** `print`, `input`, `input_int`, `input_float`, `to_int`, `to_float`, `exit`, `panic`


## Архитектура компилятора

Компилятор реализован в виде последовательного pipeline из независимых фаз:

```
source (.bl)
    │
    ▼  Lexer::      — разбиение текста на токены
    ▼  Parser::     — построение AST (рекурсивный спуск)
    ▼  Semantic::   — проверка типов и имён
    ▼  IR::         — понижение в трёхадресный код
    ▼  IR::optimize — constant folding
    ▼  Codegen::    — генерация байт-кода BL-VM
    ▼  VM::         — исполнение байт-кода
    │
bytecode (.blc)
```

Каждая фаза — отдельный модуль в своём namespace. Смешение фаз не допускается.

## Сборка

Требования: компилятор C++23 (g++ 13+ или clang++ 16+).

```bash
# Сборка через Makefile
make build

# Сборка через CMake
cmake -B build && cmake --build build
```

После сборки появятся два исполняемых файла:
- `myc` — компилятор BestLang
- `blvm` — автономный исполнитель байт-кода

## Использование

```bash
# Скомпилировать и сразу запустить
./myc prog.bl --run

# Скомпилировать в байт-код
./myc prog.bl -o prog.blc

# Запустить готовый байт-код
./blvm prog.blc

# Отладочные режимы
./myc prog.bl --dump-tokens   # вывести поток токенов
./myc prog.bl --dump-ast      # вывести AST
./myc prog.bl --dump-ir       # вывести трёхадресный IR
```

## Флаги компилятора

| Флаг | Описание |
|------|----------|
| `-o <файл>` | Путь к выходному файлу (по умолчанию: имя источника с расширением `.blc`) |
| `--run` | Скомпилировать и сразу выполнить |
| `--dump-tokens` | Вывести поток токенов и завершить |
| `--dump-ast` | Вывести AST и завершить |
| `--dump-ir` | Вывести трёхадресный IR и завершить |

## Структура проекта

```
src/          — исходный код компилятора
  lexer.*     — лексический анализатор
  parser.*    — синтаксический анализатор (рекурсивный спуск)
  semantic.*  — семантический анализатор (проверка типов)
  ir.*        — промежуточное представление (трёхадресный код)
  lowering.*  — понижение AST → IR
  optimizer.* — оптимизатор (constant folding)
  codegen.*   — генератор байт-кода BL-VM
  vm.*        — виртуальная машина BL-VM
  main.cpp    — точка входа, CLI
specs/        — спецификация языка
examples/     — примеры программ на BestLang
report.md     — отчёт о проделанной работе
build/        — директория для CMake (наполняется при сборке)
```

## Примеры

| Файл | Демонстрирует |
|------|--------------|
| `examples/hello.bl` | Hello World |
| `examples/factorial.bl` | Рекурсия, цикл while |
| `examples/structs.bl` | Структуры, массивы |
| `examples/types.bl` | Псевдонимы типов, пространства имён, приведение |
| `examples/new_features.bl` | for, динамические массивы, кортежи, impl, перегрузка, null |
| `examples/reference.bl` | Полный справочник синтаксиса |

Ошибки компилятора:
```
файл:строка:столбец: error: сообщение
```

Runtime ошибки VM:
```
runtime error: сообщение at line N
```

# Лексика языка BestLang

## 1. Общие положения

BestLang — статически типизированный императивный язык с синтаксисом,
вдохновлённым Python и Java.

Исходный текст интерпретируется как последовательность байт.
Идентификаторы и ключевые слова — только ASCII-символы.
Строковые литералы поддерживают UTF-8: допускаются любые байтовые последовательности.

Лексический анализатор преобразует исходный текст в поток токенов.
Пробельные символы и комментарии в поток токенов не передаются.

---

## 2. Базовые классы символов

```
LETTER   ::= [A-Za-z_]
DIGIT    ::= [0-9]
HEX      ::= [0-9A-Fa-f]
WS       ::= [ \t\r\n]+
NONQUOTE ::= любой ASCII-символ, кроме '"' и '\n'
ESCAPE   ::= '\' ('n' | 't' | 'r' | '"' | '\' | '0' | 'x' HEX HEX)
```

---

## 3. Комментарии

```
LINE_COMMENT  ::= '//' [^\n]*
BLOCK_COMMENT ::= '/*' (любые символы) '*/'
```

Однострочный комментарий продолжается до конца строки.
Многострочный комментарий заключается между `/*` и `*/`.
Оба вида игнорируются лексером.

---

## 4. Идентификаторы

```
IDENT ::= LETTER (LETTER | DIGIT)*
```

Идентификатор не может совпадать с ключевым словом.

Рекомендуемые соглашения:
- переменные и функции: `snake_case`
- типы и структуры: `PascalCase`

---

## 5. Ключевые слова

```
var   val   fun   return  if    else
while for   in    break   continue
struct impl type namespace
as    true  false  null   void   and   or    not
```

Все ключевые слова зарезервированы и не могут использоваться как идентификаторы.

---

## 6. Литералы

### 6.1. Целые литералы

```
INT_LIT    ::= DEC_LIT INT_SUFFIX?
             | HEX_LIT INT_SUFFIX?
             | BIN_LIT INT_SUFFIX?

DEC_LIT    ::= '0' | [1-9][0-9_]*
HEX_LIT    ::= '0x' HEX (HEX | '_')*
BIN_LIT    ::= '0b' [01] ([01] | '_')*

INT_SUFFIX ::= 'i8' | 'i16' | 'i32' | 'i64'
             | 'u8' | 'u16' | 'u32' | 'u64'
```

Символ `_` допускается как разделитель разрядов (не в начале и не в конце).

Тип без суффикса — `int32` по умолчанию.

### 6.2. Вещественные литералы

```
FLOAT_LIT    ::= DEC_LIT '.' DIGIT* EXP_PART? FLOAT_SUFFIX?
               | DEC_LIT EXP_PART FLOAT_SUFFIX?
EXP_PART     ::= ('e' | 'E') ('+' | '-')? DIGIT+
FLOAT_SUFFIX ::= 'f32' | 'f64'
```

Тип без суффикса — `float64` по умолчанию.

### 6.3. Булевы литералы

```
BOOL_LIT ::= 'true' | 'false'
```

### 6.4. Строковые литералы

```
STRING_LIT ::= '"' (UTF8_CHAR | ESCAPE)* '"'
```

Поддерживаемые escape-последовательности: `\n` `\t` `\r` `\"` `\\` `\0` `\xHH`.
Строки поддерживают UTF-8: можно писать текст на русском и других языках.

### 6.5. Null литерал

```
NULL_LIT ::= 'null'
```

Используется для инициализации и присваивания nullable-переменных (`T?`).

---

## 7. Операторы

### 7.1. Арифметические
`+   -   *   /   %`

### 7.2. Логические
`and   or   not`

### 7.3. Сравнения
`==   !=   <   <=   >   >=`

### 7.4. Прочие
`=   as   .   ..   ::   :   ?`

### 7.5. Разделители
`(  )   [  ]   {  }   ,   ;`

---

## 8. Приоритет операторов

От наименьшего к наибольшему:

| Уровень | Операторы                         | Ассоциативность |
|---------|-----------------------------------|-----------------|
| 1       | `as`                              | левая           |
| 2       | `or`                              | левая           |
| 3       | `and`                             | левая           |
| 4       | `not`                             | правая (унарный)|
| 5       | `==` `!=` `<` `>` `<=` `>=`      | левая           |
| 6       | `+` `-`                           | левая           |
| 7       | `*` `/` `%`                       | левая           |
| 8       | унарный `-`                       | правая          |
| 9       | `[i]` `.field` `ns::id` `(args)`  | левая           |

---

## 9. Разделение токенов

Лексер использует правило максимального совпадения (maximal munch).

Примеры:
- `>=` — один токен, не `>` и `=`
- `==` — один токен, не два `=`

---

## 10. Ошибки лексического анализа

Лексической ошибкой является:
- не-ASCII символ в исходном тексте
- незавершённый строковый литерал
- недопустимая escape-последовательность
- `_` в начале или конце числового литерала

При лексической ошибке компилятор завершает анализ и выводит сообщение:

```
<файл>:<строка>:<столбец>: error: <сообщение>
```

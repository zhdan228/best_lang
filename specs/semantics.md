# Семантика языка BestLang

## 1. Общая модель языка

BestLang — статически типизированный императивный язык с лексической областью
видимости и обязательной инициализацией переменных.

Базовые свойства:
- значения передаются и присваиваются по значению (value semantics)
- переменные существуют в пределах блока, в котором объявлены
- функции могут иметь побочные эффекты (ввод/вывод)
- на верхнем уровне допускаются только объявления

---

## 2. Стратегия вычисления

В языке используется call-by-value:
- аргументы вычисляются перед вызовом функции
- аргументы передаются по значению (копированием)
- операнды и аргументы вычисляются слева направо

В вызове `f(a(), b())` сначала вычисляется `a()`, затем `b()`, затем `f`.

---

## 3. Значения и связывания

### 3.1. var

`var` вводит мутабельную переменную. Переприсваивание допустимо.

### 3.2. val

`val` вводит иммутабельную переменную. Переприсваивание — ошибка компиляции.

### 3.3. Обязательная инициализация

Каждая переменная обязана быть инициализирована при объявлении.
Язык не предоставляет значений по умолчанию.
Использование переменной до объявления — ошибка компиляции.

### 3.4. Shadowing

Перекрытие имён допускается: внутреннее объявление скрывает внешнее.
Компилятор выдаёт предупреждение.
Повторное объявление имени в одной области видимости — ошибка компиляции.

---

## 4. Время жизни переменных

Переменная существует с момента объявления до конца блока `{ }`,
в котором она объявлена.

```
{
    var x: int32 = 10;
}
print(x); // ошибка компиляции: x вне области видимости
```

---

## 5. Разрешение имён

Язык использует лексическую область видимости (lexical scoping).

Правила:
- идентификатор разрешается в ближайшей охватывающей области
- функции верхнего уровня доступны в произвольном порядке (взаимная рекурсия разрешена)
- локальные переменные должны быть объявлены до использования
- доступ к элементам пространств имён через оператор `::`

---

## 6. Семантика присваивания

`lvalue = expr` вычисляет `expr` и копирует результат в `lvalue`.
Для структур и массивов — глубокое копирование всех полей и элементов.

```
var a: int32 = 5;
var b: int32 = a;
b = 10;
// a == 5
```

---

## 7. Функции

### 7.1. Параметры

Параметры — иммутабельные локальные копии. Изменение параметра не влияет
на переменную вызывающего контекста.

### 7.2. Возвращаемое значение

Функция возвращает значение через `return`.
Все ветки функции с типом возврата, отличным от `void`, обязаны содержать `return`.
Проверяется семантическим анализатором.

### 7.3. Рекурсия

Функции могут быть рекурсивными. Функции верхнего уровня могут ссылаться
друг на друга в произвольном порядке.

---

## 8. Семантика выражений

### 8.1. Арифметика

- Применима к числовым типам (`intN`, `uintN`, `float32`, `float64`).
- Оба операнда обязаны иметь одинаковый тип; иначе — ошибка компиляции.
- Деление целых: усечение к нулю. `-7 / 2 = -3`.
- Остаток: знак совпадает со знаком делимого. `-7 % 2 = -1`.
- Целочисленное переполнение: wrap-around (двоичное дополнение), поведение гарантировано.

### 8.2. Логические операции

- `and` и `or` — ленивые (short-circuit): правый операнд не вычисляется, если результат определён.
- Применимы только к `bool`.

### 8.3. Сравнения

- Применимы к числовым типам и `string`.
- Строки сравниваются лексикографически побайтово.
- Результат — `bool`.
- Нецепочечные: `a < b < c` — ошибка компиляции.

### 8.4. Приведение типов

Явное приведение: `expr as T`. Допустимые приведения:

| От → К | Допустимо |
|--------|-----------|
| intN → intM | да (усечение или расширение) |
| intN → floatM | да |
| floatM → intN | да (усечение к нулю; NaN/Inf → runtime error) |
| float32 ↔ float64 | да |

## 9. Семантика инструкций

### 9.1. if

Условие — тип `bool`. Ветка `else` необязательна.

### 9.2. while

Условие — тип `bool`, проверяется перед каждой итерацией.

### 9.3. break / continue

`break` — немедленно завершает ближайший охватывающий цикл.
`continue` — переходит к проверке условия ближайшего цикла.
Использование вне цикла — ошибка компиляции.

### 9.4. return

В функции с типом возврата, отличным от `void` — обязан содержать выражение.
В `void`-функции — без выражения.

### 9.5. for range

```
for i in start..end { тело }
```

Переменная `i` проходит значения `start, start+1, ..., end-1` (конец не включается).
Переменная объявляется автоматически с типом `int32`.

### 9.6. for C-style

```
for var i: int32 = 0; i < n; i = i + 1 { тело }
```

Аналог `while` с инициализацией и шагом в заголовке.

---

## 10. Дополнительные возможности

### 10.1. Неявное расширение типов

При бинарной операции с операндами разных числовых типов меньший тип
автоматически расширяется до большего:

```
int8 → int16 → int32 → int64 → float64
float32 → float64
```

Примеры: `int8 + int32` → `int32`; `int32 + float64` → `float64`.

### 10.2. Nullable типы

Тип `T?` означает «значение типа T или `null`».
Присваивание `T` в `T?` допустимо. Сравнение с `null` через `==` и `!=`.

```
var x: int32? = null;
x = 42;
if x != null { ... }
```

### 10.3. Перегрузка функций

Разрешено объявлять несколько функций с одинаковым именем, если они различаются
типами параметров. Нужная версия выбирается на этапе компиляции.

### 10.4. Методы структур

Блок `impl TypeName { ... }` добавляет методы к структуре.
Первый параметр метода — `self: TypeName`. Вызов: `obj.method(args)`.

## 11. Встроенные функции

### 11.1. print

`print(x)` выводит значение `x` в stdout с переводом строки.
`print(x, end="...")` — вместо перевода строки выводит указанный конец.
Принимает: `intN`, `uintN`, `float32`, `float64`, `bool`, `string`.
Возвращает: `void`.

### 11.2. input

`input()` читает строку из stdin до `\n`.
`input("подсказка")` — выводит подсказку перед чтением.
Возвращает: `string`.

### 11.3. input_int / input_float

`input_int()` или `input_int("подсказка")` — читает и преобразует в `int32`.
`input_float()` или `input_float("подсказка")` — читает и преобразует в `float64`.
При невозможности преобразования — runtime error.

### 11.4. to_int / to_float

`to_int(s: string): int32` — преобразует строку в целое число.
`to_float(s: string): float64` — преобразует строку в вещественное число.
При невозможности преобразования — runtime error.

### 11.5. exit

`exit(code: int32)` завершает программу с кодом `code`. Терминирующая операция.

### 11.6. panic

`panic(message: string)` выводит сообщение в stderr и завершает программу.
Терминирующая операция.

Формат:
```
runtime error: <message> at line <N>
```

---

## 11. Ошибки времени выполнения

При runtime error выполнение прекращается немедленно.
Сообщение выводится в stderr:

```
runtime error: <сообщение> at line <N>
```

| Ситуация                         | Сообщение                              |
|----------------------------------|----------------------------------------|
| Деление на ноль                  | `division by zero`                     |
| Выход за границы массива         | `index out of bounds: index N, size M` |
| Приведение float → int при NaN   | `invalid cast: NaN or Inf to integer`  |
| Вызов `panic`                    | сообщение пользователя                 |

---

## 12. Точка входа

Программа обязана содержать:

```
fun main(): int32 { ... }
```

Функция `main` объявляется на верхнем уровне, не принимает аргументов
и возвращает код завершения процесса.

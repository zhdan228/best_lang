# Система типов BestLang

## 1. Встроенные типы

| Тип      | Размер  | Описание                   |
|----------|---------|----------------------------|
| `int8`   | 8 бит   | Знаковое целое             |
| `int16`  | 16 бит  | Знаковое целое             |
| `int32`  | 32 бит  | Знаковое целое (по умолч.) |
| `int64`  | 64 бит  | Знаковое целое             |
| `uint8`  | 8 бит   | Беззнаковое целое          |
| `uint16` | 16 бит  | Беззнаковое целое          |
| `uint32` | 32 бит  | Беззнаковое целое          |
| `uint64` | 64 бит  | Беззнаковое целое          |
| `float32`| 32 бит  | IEEE 754 float             |
| `float64`| 64 бит  | IEEE 754 double (по умолч.)|
| `bool`   | —       | Логическое значение        |
| `string` | —       | Строка (UTF-8)             |
| `void`   | —       | Отсутствие значения        |

## 2. Составные типы

### Массивы фиксированного размера
```
[T; N]   — массив из N элементов типа T
```
Размер — часть типа. `[int32; 4]` и `[int32; 8]` — разные типы.

### Динамические массивы
```
[T]   — массив элементов типа T с изменяемым размером
```
Методы: `.push(x)`, `.pop()`, `.len()`, `.get(i)`.
Размер определяется в рантайме, изменяется методами.

### Структуры
```
struct Name { field: Type; ... }
```
Именованный составной тип с фиксированными полями.
Методы добавляются через блок `impl`.

### Кортежи
```
(T1, T2, ...)   — фиксированный набор значений разных типов
```
Доступ к элементам: `.0`, `.1`, `.2`, ...
Кортеж из одного элемента не допускается (используй просто значение).

### Nullable типы
```
T?   — значение типа T или null
```
Значение `null` — специальный маркер отсутствия значения.
Присваивание `T` в `T?` допустимо без явного приведения.
Проверка: `x == null` или `x != null`.

## 3. Правила типизации

- **Номинальная типизация** — два типа совместимы только если их имена совпадают.
- **Неявное расширение** — меньший числовой тип автоматически расширяется до большего
  при бинарных операциях: `int8 → int32 → int64 → float64`.
- **Явное приведение** — для всех остальных преобразований используется `as`.
- **Value semantics** — все значения передаются и присваиваются по значению.

## 4. Неявное расширение числовых типов

При бинарной операции или присваивании компилятор автоматически вставляет
приведение если один тип является «шире» другого:

```
int8 → int16 → int32 → int64 → float64
float32 → float64
```

Примеры:
```
var a: int8  = 100i8;
var b: int32 = 200;
var c = a + b;    // c: int32 = 300 (int8 расширяется до int32)

var n: int32   = 7;
var f: float64 = 0.5;
var r = n + f;    // r: float64 = 7.5 (int32 расширяется до float64)
```

## 5. Допустимые явные приведения (`as`)

| От          | К           | Поведение                         |
|-------------|-------------|-----------------------------------|
| `intN`      | `intM`      | Усечение или знаковое расширение  |
| `uintN`     | `uintM`     | Усечение или нулевое расширение   |
| `intN`      | `floatM`    | Целое → вещественное              |
| `floatM`    | `intN`      | Усечение к нулю; NaN/Inf → ошибка |
| `float32`   | `float64`   | Расширение точности               |
| `float64`   | `float32`   | Сужение точности                  |

## 6. Синонимы типов

Синоним вводит альтернативное имя для существующего типа:
```
type Meters = float64;
type Board  = [int32; 64];
```
Синоним не создаёт новый тип — `Meters` и `float64` взаимозаменяемы.

## 7. Вывод типов

Если аннотация типа опущена при объявлении переменной, тип выводится из инициализатора:
```
var x = 42;       // int32
var y = 3.14;     // float64
var s = "hello";  // string
var b = true;     // bool
```

Тип возврата функции указывается явно — вывод типа возврата не поддерживается.

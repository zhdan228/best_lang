# Генерация кода BestLang

## 1. Целевая платформа

- собственная стековая виртуальная машина BL-VM
- байт-код формата `.blc`
- реализация VM на C++23

---

## 2. Этапы компиляции

```
source (.bl) → Lexer → Parser → SemanticAnalyzer → Lowering → Optimizer → Codegen → bytecode (.blc)
```

- исходный текст преобразуется в поток токенов (`Lexer::`)
- поток токенов → в AST (`Parser::`)
- AST проходит семантический анализ и аннотируется типами (`Semantic::`)
- аннотированный AST понижается в IR — трёхадресный код (`IR::`)
- IR проходит оптимизацию (`IR::`)
- оптимизированный IR транслируется в байт-код BL-VM (`Codegen::`)

Каждая фаза — отдельный модуль в своём namespace. Смешение фаз не допускается.

---

## 3. Промежуточное представление (IR)

IR — трёхадресный код. Каждая инструкция содержит не более одного оператора
и не более трёх операндов.

```
t0 = a + b
t1 = -a
t2 = a[i]
a[i] = t2
t3 = a.field
a.field = t3
t4 = call f(a, b)
if t4 goto L1
goto L2
L1:
return t4
```

IR поддерживает вывод в читаемом виде через флаг `--dump-ir`.

### 3.1. Оптимизация: Constant Folding

Выражения, оба операнда которых являются константами, вычисляются на этапе компиляции:

```
t0 = 2 + 3          →  t0 = 5
t1 = true and false  →  t1 = false
```

## 4. Формат файла байт-кода (.blc)

```
[Заголовок]
[Пул констант]
[Секция функций]
[Секция глобальных переменных]
[Таблица отладочных строк]
```

### 4.1. Заголовок

| Смещение | Размер | Поле                             |
|----------|--------|----------------------------------|
| 0        | 2 байт | Магическое число `0x424C` ("BL") |
| 2        | 1 байт | Версия формата `0x01`            |
| 3        | 1 байт | Зарезервировано `0x00`           |
| 4        | 4 байт | Смещение пула констант           |
| 8        | 4 байт | Смещение секции функций          |
| 12       | 4 байт | Смещение таблицы отладки         |
| 16       | 4 байт | Индекс функции `main`            |
| 20       | 4 байт | Смещение секции глобальных перем.|

Все многобайтовые поля — little-endian.

### 4.2. Пул констант

Каждый элемент: тег типа (1 байт) + значение.

| Тег    | Тип              |
|--------|------------------|
| `0x01` | `int64`          |
| `0x02` | `uint64`         |
| `0x03` | `float64`        |
| `0x04` | `bool`           |
| `0x05` | `string` (ASCII) |
| `0x06` | Имя функции      |

### 4.3. Секция функций

Каждая функция содержит: имя (индекс в пуле), число параметров,
число слотов локальных переменных, последовательность инструкций.

### 4.4. Секция глобальных переменных

Число глобальных переменных (4 байта) + индекс инициализатора в пуле констант (2 байта на каждую).

---

## 5. Представление значений во время выполнения

| Тип BestLang       | Представление в VM                            |
|--------------------|-----------------------------------------------|
| `int8`…`int64`     | Знаковое 64-битное целое (знаковое расширение)|
| `uint8`…`uint64`   | Беззнаковое 64-битное целое                   |
| `float32`          | IEEE 754 float, хранится как double           |
| `float64`          | IEEE 754 double                               |
| `bool`             | Целое: 0 (`false`) или 1 (`true`)             |
| `string`           | Указатель на внутренний объект строки         |
| `[T; N]`           | Указатель на внутренний объект массива        |
| struct             | Указатель на внутренний объект структуры      |
| `void`             | Маркер UNIT без полезной нагрузки             |

## 6. Таблица опкодов BL-VM

Опкод — 1 байт. Операнды следуют за опкодом, little-endian.

### Константы

| Опкод  | Мнемоника    | Операнд      | Описание                            |
|--------|--------------|--------------|-------------------------------------|
| `0x01` | `PUSH_CONST` | `uint16` idx | Поместить константу из пула на стек |
| `0x02` | `PUSH_TRUE`  | —            | Поместить `true`                    |
| `0x03` | `PUSH_FALSE` | —            | Поместить `false`                   |
| `0x04` | `PUSH_UNIT`  | —            | Поместить void-маркер               |

### Локальные переменные

| Опкод  | Мнемоника    | Операнд      | Описание                               |
|--------|--------------|--------------|----------------------------------------|
| `0x10` | `LOAD`       | `uint16` idx | Загрузить локальную переменную на стек |
| `0x11` | `STORE`      | `uint16` idx | Снять вершину стека, сохранить в слот  |
| `0x12` | `LOAD_GLOB`  | `uint16` idx | Загрузить глобальную переменную        |
| `0x13` | `STORE_GLOB` | `uint16` idx | Сохранить в глобальную переменную      |

### Арифметика

| Опкод  | Мнемоника | Описание                                        |
|--------|-----------|-------------------------------------------------|
| `0x20` | `IADD`    | Сложение целых (int64)                          |
| `0x21` | `ISUB`    | Вычитание целых                                 |
| `0x22` | `IMUL`    | Умножение целых                                 |
| `0x23` | `IDIV`    | Деление целых (деление на ноль → runtime error) |
| `0x24` | `IMOD`    | Остаток от деления                              |
| `0x25` | `INEG`    | Унарный минус (целые)                           |
| `0x28` | `FADD`    | Сложение вещественных (float64)                 |
| `0x29` | `FSUB`    | Вычитание вещественных                          |
| `0x2A` | `FMUL`    | Умножение вещественных                          |
| `0x2B` | `FDIV`    | Деление вещественных                            |
| `0x2C` | `FNEG`    | Унарный минус (вещественные)                    |

### Логика

| Опкод  | Мнемоника | Описание            |
|--------|-----------|---------------------|
| `0x30` | `BAND`    | Логическое И (bool) |
| `0x31` | `BOR`     | Логическое ИЛИ      |
| `0x32` | `BNOT`    | Логическое НЕ       |

### Сравнения

| Опкод  | Мнемоника | Описание                        |
|--------|-----------|---------------------------------|
| `0x40` | `IEQ`     | Равенство целых                 |
| `0x41` | `INEQ`    | Неравенство целых               |
| `0x42` | `ILT`     | Меньше (целые)                  |
| `0x43` | `IGT`     | Больше (целые)                  |
| `0x44` | `ILTE`    | Меньше или равно (целые)        |
| `0x45` | `IGTE`    | Больше или равно (целые)        |
| `0x48` | `FEQ`     | Равенство вещественных          |
| `0x49` | `FNEQ`    | Неравенство вещественных        |
| `0x4A` | `FLT`     | Меньше (вещественные)           |
| `0x4B` | `FGT`     | Больше (вещественные)           |
| `0x4C` | `FLTE`    | Меньше или равно (вещественные) |
| `0x4D` | `FGTE`    | Больше или равно (вещественные) |
| `0x4E` | `SEQ`     | Равенство строк                 |
| `0x4F` | `SNEQ`    | Неравенство строк               |

### Управление потоком

| Опкод  | Мнемоника   | Операнд      | Описание                                      |
|--------|-------------|--------------|-----------------------------------------------|
| `0x50` | `JMP`       | `int16` off  | Безусловный переход (смещение от текущего PC) |
| `0x51` | `JMP_FALSE` | `int16` off  | Переход если вершина стека == false           |
| `0x52` | `JMP_TRUE`  | `int16` off  | Переход если вершина стека == true            |
| `0x53` | `CALL`      | `uint16` idx | Вызов функции (индекс в таблице функций)      |
| `0x54` | `RET`       | —            | Возврат из void-функции                       |
| `0x55` | `RET_VAL`   | —            | Возврат значения с вершины стека              |

### Стек

| Опкод  | Мнемоника | Описание                  |
|--------|-----------|---------------------------|
| `0x60` | `POP`     | Снять вершину стека       |
| `0x61` | `DUP`     | Дублировать вершину стека |

### Массивы

| Опкод  | Мнемоника    | Операнд      | Описание                        |
|--------|--------------|--------------|---------------------------------|
| `0x70` | `NEW_ARRAY`  | `uint16` len | Создать массив длиной len       |
| `0x71` | `ARRAY_GET`  | —            | `[arr, idx] → [value]`          |
| `0x72` | `ARRAY_SET`  | —            | `[arr, idx, val] → []`          |
| `0x73` | `ARRAY_LEN`  | —            | `[arr] → [int64]`               |
| `0x74` | `ARRAY_PUSH` | —            | `[arr, val] → []` (динамич.)    |
| `0x75` | `ARRAY_POP`  | —            | `[arr] → [val]` (динамич.)      |

`NEW_ARRAY 0` создаёт пустой динамический массив (используется для `[T]`).

### Структуры

| Опкод  | Мнемоника    | Операнд      | Описание                    |
|--------|--------------|--------------|-----------------------------|
| `0x80` | `NEW_STRUCT` | `uint16` n   | Создать структуру с n полей |
| `0x81` | `FIELD_GET`  | `uint16` idx | Загрузить поле по индексу   |
| `0x82` | `FIELD_SET`  | `uint16` idx | Записать поле по индексу    |

### Строки

| Опкод  | Мнемоника    | Описание                         |
|--------|--------------|----------------------------------|
| `0x90` | `STR_CONCAT` | Конкатенация двух строк на стеке |
| `0x91` | `STR_LEN`    | `[str] → [uint64]`              |

### Приведение типов

| Опкод  | Мнемоника | Описание                                            |
|--------|-----------|-----------------------------------------------------|
| `0xA0` | `I2F`     | `int64 → float64`                                   |
| `0xA1` | `F2I`     | `float64 → int64`, усечение; NaN/Inf → runtime error|
| `0xA2` | `F32_F64` | `float32 → float64`                                 |
| `0xA3` | `F64_F32` | `float64 → float32`                                 |
| `0xA4` | `ITRUNC`  | `uint8` bits: усечение целого до N бит              |
| `0xA5` | `IEXT_S`  | `uint8` bits: знаковое расширение до N бит          |
| `0xA6` | `IEXT_U`  | `uint8` bits: беззнаковое расширение до N бит       |

### Встроенные функции

| Опкод  | Мнемоника     | Описание                                              |
|--------|---------------|-------------------------------------------------------|
| `0xB0` | `PRINT`       | `[val] → []` Вывести в stdout + `\n`                  |
| `0xB1` | `INPUT`       | `[] → [str]` Прочитать строку из stdin                |
| `0xB2` | `EXIT`        | `[code] → ` Завершить программу с кодом              |
| `0xB3` | `PANIC`       | `[msg] → ` Аварийное завершение с сообщением         |
| `0xB4` | `PRINT_END`   | `[val, end_str] → []` Вывести val + end_str (без `\n`)|
| `0xB5` | `INPUT_INT`   | `[] → [int32]` Прочитать и распарсить как int         |
| `0xB6` | `INPUT_FLOAT` | `[] → [float64]` Прочитать и распарсить как float     |
| `0xB7` | `TO_INT`      | `[str] → [int32]` Преобразовать строку в int          |
| `0xB8` | `TO_FLOAT`    | `[str] → [float64]` Преобразовать строку в float      |

`PRINT_END` используется для `print(x, end="...")` — аналог Python `end=` параметра.

## 7. Соглашения о вызовах BL-VM

- Вызывающая сторона помещает аргументы на стек слева направо перед `CALL`.
- `CALL` создаёт новый стейм вызова: указатель на предыдущий стейм, return PC,
  слоты локальных переменных.
- Параметры занимают слоты с индексами `0, 1, …, N-1`.
- При `RET_VAL` возвращаемое значение помещается на стек вызывающего стейма.
- При `RET` стек вызывающего не изменяется.

---

## 8. Ошибки времени выполнения

При runtime error VM прекращает исполнение и выводит в stderr:

```
runtime error: <сообщение> at line <N>
```

Номер строки извлекается из таблицы отладочных строк файла `.blc`.

Минимально обрабатываемые ошибки:
- деление на ноль
- выход за границы массива
- приведение NaN/Inf к целому типу
